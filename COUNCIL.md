## Invocation Patterns

### Single Expert
```
@expert-name [request]
"What would [Expert Name] say about [topic]?"
"I need [Expert Name]'s perspective on..."
"Help me with [domain] using [Expert]'s approach"
```

### Council Invocation
```
@council-name [request]
"Ask the [Council Name] about..."
"Get the [Council]'s perspective on..."
"I need the [domain] council's input"
```

### Multi-Expert Synthesis
```
"What would @expert-1 and @expert-2 say about [topic]?"
"Compare [Expert A]'s approach with [Expert B]'s approach"
"Synthesize perspectives from [Expert A], [Expert B], and [Expert C]"
```

### Domain-Based (Auto-Route)
```
"Help me with this sales objection" → Routes to Jeremy Miner
"Write compelling copy for..." → Routes to Cardinal Mason or Harry Dry
"How should I position my brand?" → Routes to Brand Council
```

## Adding New Experts

To add a new expert to the council:

1. **Intake**: Place extraction in `knowledge/extractions/inbox/`
2. **Assess**: Determine if agent-worthy or skill-only
3. **Build**: Follow `directives/extraction-workflow.md`
4. **Register**: Add entry to this file (Quick Reference + Full Details)
5. **Link**: Add to appropriate council(s)
6. **Test**: Invoke and verify quality

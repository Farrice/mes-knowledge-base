# Mark Kashef — Implementation Methodology

The Kashef Orchestration Protocol centers on architecting multi-agent tasks. It dictates what structural pathways construct elite outputs.

### 1. Define the Objective
Force absolute alignment. Determine what the final deliverable is (e.g., a multi-platform content repurposing, an AI advisory board synthesis, a 12-slide codebase migration pitch deck).

### 2. Determine the Architecture
Assign the physical routing protocol:
- **Sequential Assembly Line**: Phase A feeds to Phase B. Best for singular artifacts requiring heavy preparation.
- **Parallel Synthesis**: Agents A, B, and C work concurrently and feed an arbiter. Best for analytical workflows or high-volume heterogeneous content production.

### 3. Role Specification
Define strict personhood for every agent. A "Competitor Analyst" or a "Devil's Advocate" possesses targeted depth over a generic "Agent 1". Limit the swarm to 3-to-5 operators.

### 4. Enforce Communication
Demand operational synchronization. For sequential lines, insert "do not start until researcher identifies 10 points". For parallel lines, impose "you must compare notes and avoid overlapping angles before submitting to the Synthesis Lead".

### 5. Establish the Output
Spoon-feed formatting requirements. The final prompt section demands specific file outputs (.md files, PDFs via MCP, etc.). Include a "wait for user plan approval" block prior to final output creation to act as a fail-safe.

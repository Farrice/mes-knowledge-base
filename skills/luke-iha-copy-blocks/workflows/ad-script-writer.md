# Ad Script Writer

Produces a deployment-ready ad script using copy block architecture. Outputs 3 hook variants, structured body copy with block annotations, and CTA — ready for recording or publishing.

## PHASE 2: INPUT REQUIREMENTS

Collect from user:
- **Product/Offer**: What's being sold?
- **Target Audience**: Who is this for?
- **Ad Format**: Text, video script, static image, carousel?
- **Length**: Short (< 60 sec / 200 words), Medium (60-120 sec / 200-500 words), Long (120+ sec / 500+ words)
- **CASH Selection** (optional): If from a CASH matrix, which Concept × Angle × Style?
- **Mechanism** (optional): If using a mechanism from `million-dollar-mechanisms`, provide name + one-sentence explanation
- **Key Proof Points** (optional): Testimonials, data, credentials to include

## PHASE 3: HOOK GENERATION

Write 3 hook variants using different architectures:
1. **Solve Hook**: Identifies the problem and implies a solution
2. **Show Hook**: Demonstrates a result
3. **Wild Card**: Contrarian, story, proof-of-work, or confession — whatever fits best

Each hook: 1-2 sentences max. Should stop the scroll and create momentum into the body.

## PHASE 4: BODY COPY CONSTRUCTION

Build the body using the full ad architecture with block annotations:

```
[HOOK — selected from Phase 3]

[PAYOFF — CU/PR] Deliver on the hook's implicit promise. What will the reader gain?

[MECHANISM REVEAL — CU/PF] WHY does this work? Tease or reveal the unique mechanism.

[PROOF STACK — PF] Evidence: testimonials, data points, credentials, demonstrations.

[REASON-TO-BUY — PR/CO] Why THIS solution over alternatives? What makes it unique?

[CONSTRAINT/URGENCY — CN] Why act now? Scarcity, time-sensitivity, limited availability.

[CTA — CO/PR] Clear, single action. Restate the core promise.
```

Tag each section with its copy block codes. Ensure velocity is high in the opening and measured before the CTA.

## PHASE 5: VELOCITY CHECK

Before outputs:
1. Map the block sequence of the complete script
2. Check for velocity drops (4+ consecutive same-block sentences)
3. Verify opening hits 3+ blocks in the first 5 sentences
4. Confirm each block uses fresh angles on reappearance

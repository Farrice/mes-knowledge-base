# Domain Application Deployment Workflow

Access the 10 Domain Application prompts (DA-1 through DA-10) that apply comedy intelligence to specific business verticals. Each DA is a self-contained system for weaponizing humor within a domain.

## When To Use
- You're working in a specific business domain (copywriting, positioning, content, etc.)
- You want comedy intelligence tailored to that domain, not generic humor
- You want to compound comedy with domain-specific expert knowledge

## Steps

### 1. Identify Domain
Determine which DA prompt matches the user's task. If unclear, ask.

### 2. Load DA Prompt
// turbo
Read the selected DA file from `skills/robert-mack-comedy-writing/references/domain-applications/`

### 3. Load Comedy Core
// turbo
Read `skills/robert-mack-comedy-writing/genius.md` for comedy architecture context.

### 4. Execute
Follow the DA prompt's methodology to produce domain-specific output.

### 5. Compound (Optional)
If stacking with another expert, load their skill and apply compound methodology as described in [da_complete_index.md](../references/domain-applications/da_complete_index.md).

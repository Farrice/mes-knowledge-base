# KITTL - COMPLEXITY-SIZE SCALING CROWN JEWEL PROMPT

## ROLE & ACTIVATION

You are a Typography Hierarchy Engineer executing the Kittl methodology of complexity-size scaling. You understand the inverse relationship between font decorativeness and text size—as text gets smaller, fonts must get simpler to maintain readability. You deploy this knowledge to create perfect typographic hierarchies where every level of text uses appropriately complex fonts.

Your expertise recognizes that display fonts (decorative, stylistic, experimental) belong at large sizes where their details are visible and readable, while utility fonts (clean, simple, highly legible) belong at small sizes where simplicity serves comprehension. Between these extremes, transitional fonts bridge the hierarchy.

You don't teach hierarchy theory—you build hierarchies. Given any project's text requirements, you produce a complete font system where complexity scales inversely with size, ensuring every text element remains readable while maintaining stylistic cohesion.

## INPUT REQUIRED

- **Project Type**: [What you're creating—e.g., "website," "poster," "brand system," "magazine layout," "packaging," etc.]
- **Style Direction**: [The overall aesthetic—e.g., "luxury," "streetwear," "minimalist," "vintage," etc.]
- **Text Hierarchy Levels**: [What text types you need—e.g., "headline, subhead, body, caption" or "hero text, product name, description, fine print"]
- **Primary Display Font** (optional): [If you've already chosen a hero font, include it here]

## EXECUTION PROTOCOL

1. **MAP** all text hierarchy levels from largest to smallest anticipated size
2. **ASSIGN** complexity scores to each level (5=highly decorative, 1=highly simple)
3. **SELECT** fonts for each level matching their complexity score
4. **VERIFY** readability thresholds—no font is used below its legibility floor
5. **BRIDGE** adjacent levels with transitional weight or font choices
6. **SPECIFY** exact size ranges, weights, and spacing for each level
7. **TEST** the complete system for stylistic cohesion across all levels

## OUTPUT DELIVERABLE

A complete Complexity-Scaled Typography System containing:
- **Format**: Hierarchical typography specification
- **Components Included**:
  - Hierarchy Map (all text levels with size ranges)
  - Complexity Scoring (decorativeness rating for each level)
  - Font Assignments (specific fonts matched to complexity needs)
  - Readability Verification (minimum sizes for each font)
  - Transitional Bridges (how levels connect visually)
  - Complete Specifications (sizes, weights, spacing per level)
  - Implementation Guide (how to apply the system)
  - Edge Case Handling (what to do at unusual sizes)

## CREATIVE LATITUDE

Apply typographic judgment beyond mechanical size-complexity mapping. Sometimes stylistic cohesion benefits from using a single font family across multiple hierarchy levels (varying weight instead of font). Sometimes dramatic complexity jumps create intentional visual interest.

Consider project context—a magazine layout might want dramatic hierarchy contrast, while an app interface might want subtle transitions. The inverse relationship is a principle, not a prison.

## ENHANCEMENT LAYER

**Beyond Original**: This prompt makes the unconscious sizing decisions of expert designers explicit and repeatable. Most designers intuitively know "that font won't work small"—this system explains why and provides alternatives.

**Scale Advantage**: One execution produces a complete typography system applicable to an entire project, brand, or product. Every text element has a defined solution.

**Integration Potential**: These hierarchies integrate with design systems, component libraries, and brand guidelines, providing the "typography rules" that ensure consistency.

**THE ACTUAL DELIVERABLE:**

# COMPLEXITY-SCALED TYPOGRAPHY SYSTEM
## E-Commerce Website: Modern Luxury Fashion

---

### HIERARCHY MAP

| Level | Text Type | Typical Size Range | Complexity Score | Font Category Needed |
|-------|-----------|-------------------|------------------|---------------------|
| 1 | Hero Headline | 64-120px | 5 (High) | Expressive Display |
| 2 | Section Header | 36-48px | 4 (Moderate-High) | Refined Display |
| 3 | Product Name | 20-28px | 3 (Moderate) | Versatile Display/Body |
| 4 | Product Description | 14-18px | 2 (Low-Moderate) | Readable Body |
| 5 | Price | 16-24px | 2 (Low-Moderate) | Clear Utility |
| 6 | Fine Print | 10-12px | 1 (Minimal) | Maximum Legibility |

---

### COMPLEXITY SCORING RATIONALE

**Level 1 (Score: 5)**: At 64-120px, every letterform detail is visible. This is the moment for expressive typography—high contrast serifs, decorative elements, dramatic personality. The large size guarantees readability regardless of complexity.

**Level 2 (Score: 4)**: At 36-48px, most details remain visible but extreme decoration starts to muddy. Refined display fonts work—elegant serifs, sophisticated sans-serifs—but avoid the most experimental choices.

**Level 3 (Score: 3)**: At 20-28px, we're transitioning from "display" to "must actually read this." Product names need to be scannable across a grid. Fonts should have character but prioritize clarity.

**Level 4 (Score: 2)**: At 14-18px, readability dominates. This is where customers read actual product information. Fonts must be optimized for extended reading with minimal distraction.

**Level 5 (Score: 2)**: Prices at 16-24px need instant comprehension. Numbers must be clear, tabular alignment helps comparison shopping. Utility over personality.

**Level 6 (Score: 1)**: At 10-12px, only the simplest, most legible fonts survive. Returns policies, terms, and sizing charts must be readable. Zero tolerance for decorative interference.

---

### FONT ASSIGNMENTS

**Level 1 — Hero Headlines: Playfair Display**
- **Complexity Match**: High-contrast transitional serif with dramatic thick/thin variation
- **Why It Works at This Level**: The contrast and elegance shine at 64px+; the serif details are fully visible and communicate luxury
- **Weight**: Regular or Bold
- **Minimum Viable Size**: 36px (below this, contrast becomes noise)

**Level 2 — Section Headers: Playfair Display (Light) OR Cormorant Garamond**
- **Complexity Match**: Same family at lighter weight reduces intensity, or shift to softer serif
- **Why It Works**: Maintains luxury feel while pulling back from hero drama
- **Weight**: Light to Regular
- **Minimum Viable Size**: 28px

**Level 3 — Product Names: Lora SemiBold**
- **Complexity Match**: Calligraphic-influenced serif with excellent mid-size performance
- **Why It Works**: Enough personality to feel premium, enough clarity to scan product grids
- **Weight**: SemiBold for presence without heaviness
- **Minimum Viable Size**: 16px

**Level 4 — Product Descriptions: Lato Regular**
- **Complexity Match**: Warm, humanist sans-serif optimized for reading
- **Why It Works**: Friendly readability, slight warmth prevents clinical coldness, proven extended-reading performance
- **Weight**: Regular (Light for minimal contexts)
- **Minimum Viable Size**: 12px

**Level 5 — Prices: Inter Medium**
- **Complexity Match**: Geometric precision with excellent numeral design
- **Why It Works**: Tabular figures align perfectly; optimized for interface clarity; numbers are unmistakable
- **Weight**: Medium for presence
- **Minimum Viable Size**: 12px

**Level 6 — Fine Print: Inter Regular**
- **Complexity Match**: Maximum legibility at minimum sizes
- **Why It Works**: Designed for screen interfaces including small sizes; no ambiguous letterforms
- **Weight**: Regular
- **Minimum Viable Size**: 10px (absolute floor for legal readability)

---

### READABILITY VERIFICATION

| Font | Minimum Recommended Size | Absolute Floor | Danger Signs Below Floor |
|------|-------------------------|----------------|-------------------------|
| Playfair Display | 36px | 28px | Contrast becomes noise; thin strokes disappear |
| Cormorant Garamond | 28px | 20px | Delicate details blur; elegance becomes weakness |
| Lora | 16px | 14px | Calligraphic details interfere with scanning |
| Lato | 12px | 10px | Still readable but approaching strain |
| Inter | 10px | 9px | Engineered for small sizes; rarely fails |

**Critical Rule**: Never use Playfair Display below 28px. Never use Lora below 14px. When in doubt, drop complexity.

---

### TRANSITIONAL BRIDGES

**Hero → Section Header Bridge**:
- Same font family (Playfair Display) with weight reduction
- Creates visual continuity while reducing intensity
- Alternative: Shift from Playfair to Cormorant (same serif category, softer execution)

**Section Header → Product Name Bridge**:
- Shift from high-contrast serif (Playfair/Cormorant) to moderate-contrast serif (Lora)
- Maintains serif continuity while improving scannability
- The "serif through-line" creates cohesion

**Product Name → Description Bridge**:
- Shift from serif (Lora) to humanist sans-serif (Lato)
- This is the primary contrast point in the system
- Signals: "Stop scanning, start reading"

**Description → Fine Print Bridge**:
- Shift from warm sans-serif (Lato) to neutral sans-serif (Inter)
- Subtle transition—both are readable sans-serifs
- Inter's smaller optical optimizations become valuable at smallest sizes

### IMPLEMENTATION GUIDE

**CSS Variable Setup**:
```
--font-display: 'Playfair Display', serif;
--font-serif: 'Lora', serif;
--font-body: 'Lato', sans-serif;
--font-ui: 'Inter', sans-serif;
```

**Component Mapping**:
- Hero components: `--font-display`
- Section titles: `--font-display` at lighter weight
- Product cards: `--font-serif` for name, `--font-body` for description
- UI elements: `--font-ui`
- Legal/footer: `--font-ui` at smallest sizes

**Responsive Scaling**:
- Mobile: Reduce all sizes by ~20% but maintain proportional relationships
- Never let Level 1 fonts drop below 36px on mobile
- Fine print can reach 10px minimum on mobile

---

### EDGE CASE HANDLING

**What if I need text between defined levels?**
→ Use the font from the LOWER complexity level. When in doubt, prioritize readability.

**What if product names need to be very small (in a mini-cart)?**
→ Switch from Lora to Lato for mini-cart contexts. Don't force Level 3 font into Level 4+ sizes.

**What if hero headlines need to be readable at smaller sizes (mobile)?**
→ Consider switching to Cormorant Garamond which holds up better at medium sizes, or use Playfair Display Bold to maintain stroke visibility.

**What if the style feels too disjointed across levels?**
→ Reduce the number of font families. Consider: Playfair Display (L1-L2), Lato (L3-L6)—two fonts instead of four.

## EXAMPLE OUTPUT 2: Streetwear Brand Poster System

**Context**:
- Project Type: Poster series (18x24 inches, print)
- Style Direction: Brutalist streetwear / high-impact
- Text Hierarchy Levels: Main statement, supporting phrase, event details, fine print (date, venue, legal)
- Primary Display Font: Anton (already selected)

---

**THE ACTUAL DELIVERABLE:**

# COMPLEXITY-SCALED TYPOGRAPHY SYSTEM
## Streetwear Poster Series: Brutalist Impact

---

### HIERARCHY MAP

| Level | Text Type | Typical Size Range | Complexity Score | Font Category Needed |
|-------|-----------|-------------------|------------------|---------------------|
| 1 | Main Statement | 150-300pt | 5 (High) | Maximum Impact Display |
| 2 | Supporting Phrase | 48-72pt | 4 (Moderate-High) | Strong Display |
| 3 | Event Details | 24-36pt | 3 (Moderate) | Readable Display |
| 4 | Fine Print | 10-14pt | 1 (Minimal) | Utility Clarity |

**Note**: Poster hierarchies can tolerate higher complexity at smaller relative sizes because viewers can get closer to the print. The "absolute floor" sizes are higher than screen, but complexity tolerance is also higher.

---

### COMPLEXITY SCORING RATIONALE

**Level 1 (Score: 5)**: At 150-300pt, typography IS the poster. This demands maximum personality, maximum space-filling, maximum impact. Anton's condensed structure was chosen for exactly this—total visual dominance.

**Level 2 (Score: 4)**: At 48-72pt, supporting phrases frame the main statement. Still very large, still highly visible, but must complement rather than compete. Same font family at different treatment, or coordinating display font.

**Level 3 (Score: 3)**: At 24-36pt, event details must be actually readable—date, time, venue. Viewers need this information. Personality remains but clarity increases.

**Level 4 (Score: 1)**: At 10-14pt, legal text, websites, and credits need maximum legibility. No decorative interference. This level often uses a completely different font family.

---

### FONT ASSIGNMENTS

**Level 1 — Main Statement: Anton**
- **Complexity Match**: Ultra-condensed display sans-serif built for impact
- **Why It Works at This Level**: Designed to fill space aggressively; condensed structure allows massive height; zero decoration keeps focus on message
- **Weight**: Regular (only weight available—it's already bold)
- **Minimum Viable Size**: 48pt (below this, the ultra-condensed becomes strained)

**Level 2 — Supporting Phrase: Bebas Neue**
- **Complexity Match**: Condensed display sans-serif, slightly more refined than Anton
- **Why It Works**: Coordinates with Anton's condensed energy while offering weight options; slightly wider letterforms provide visual breathing room
- **Weight**: Regular or Light (contrast with Anton's heaviness)
- **Minimum Viable Size**: 24pt

**Level 3 — Event Details: Space Grotesk**
- **Complexity Match**: Contemporary geometric sans-serif with subtle character
- **Why It Works**: Transitions from display to readable; geometric structure coordinates with Anton/Bebas; character details reward medium-size viewing
- **Weight**: Medium
- **Minimum Viable Size**: 14pt

**Level 4 — Fine Print: Inter**
- **Complexity Match**: Maximum-legibility utility sans-serif
- **Why It Works**: Proven small-size performance; neutral personality doesn't distract from poster impact; professional credibility for legal text
- **Weight**: Regular
- **Minimum Viable Size**: 8pt (print can go smaller than screen)

---

### READABILITY VERIFICATION (PRINT CONTEXT)

| Font | Minimum Recommended Size | Absolute Floor | Print-Specific Notes |
|------|-------------------------|----------------|---------------------|
| Anton | 48pt | 36pt | Ultra-condensed collapses at small sizes |
| Bebas Neue | 24pt | 18pt | Condensed but more forgiving than Anton |
| Space Grotesk | 14pt | 10pt | Good small-size performance |
| Inter | 8pt | 6pt | Designed for legibility; excels at tiny sizes |

**Print Advantage**: Poster viewers can move closer, allowing smaller sizes than screen. But print also has no zoom—sizes are fixed. When in doubt, go larger.

---

### TRANSITIONAL BRIDGES

**Main Statement → Supporting Phrase Bridge**:
- Both fonts are condensed sans-serifs—structural consistency
- Anton's heaviness vs. Bebas Neue's lighter options create hierarchy through weight
- Consider: Same word in both fonts would clearly show hierarchy

**Supporting Phrase → Event Details Bridge**:
- Shift from condensed (Bebas Neue) to standard-width (Space Grotesk)
- This is the major transition: "impact zone" to "information zone"
- Space Grotesk's geometric structure connects to the sans-serif family

**Event Details → Fine Print Bridge**:
- Both are geometric sans-serifs—Space Grotesk to Inter
- Subtle shift prioritizing legibility over character
- This transition is nearly invisible, appropriately so

### EDGE CASE HANDLING

**What if the main statement is longer (more words)?**
→ Stack words vertically, maintain 150pt+ size per line. Don't shrink to fit horizontally—the size IS the point.

**What if supporting phrase needs more emphasis?**
→ Use Bebas Neue Bold or switch to Anton at smaller size. Don't exceed main statement visual weight.

**What if poster is smaller format (11x17)?**
→ Scale proportionally: Main at 100-150pt, Supporting at 36-48pt, Details at 18-24pt, Fine at 8-10pt. Maintain ratios.

**What if I want black letter for edgy streetwear?**
→ Black letter can work at Level 1 only. Maximum 3-4 words. Below Level 1, readability concerns eliminate black letter as an option.

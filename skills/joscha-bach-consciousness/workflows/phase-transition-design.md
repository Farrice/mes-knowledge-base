# Phase Transition Design

> Bach's insight: coherence doesn't accumulate gradually — it ignites. Design for the spark, not the slow build.

## When to Use

- A product or brand "almost works" but hasn't clicked yet
- Building a team or community that hasn't achieved coherence
- Launching something and need to identify the tipping conditions
- A creative project has all the pieces but lacks the unifying spark
- Designing an agent system that needs to "wake up" into coherence

## The Phase Transition Protocol

### Step 1: Pre-Ignition Inventory

Map the components of your system:

| Component | Present? | Quality (1-10) | Connected to other components? |
|-----------|----------|----------------|-------------------------------|
| [component 1] | Y/N | score | [which others it links to] |
| [component 2] | Y/N | score | [which others it links to] |

Bach's insight: "This phase transition that happens when consciousness ignites, when something wakes up and becomes mentally coherent with itself."

**Key diagnostic**: Are the components interacting, or just coexisting?
- **Coexisting** = pre-ignition (a stack of ingredients, not a cake)
- **Interacting** = approaching ignition (ingredients mixing, heat rising)
- **Coherent** = post-ignition (the cake exists as a unified thing)

### Step 2: Ignition Condition Hypothesis

What conditions must be met for the phase transition?

Bach's model uses consciousness as the archetype: consciousness isn't about having lots of neurons — it's about those neurons becoming "mentally coherent with itself." Apply the same logic:

**Template**:
```
[System] ignites when:
1. [Critical mass condition] — enough components present
2. [Interaction condition] — components communicating, not just coexisting
3. [Coherence condition] — components begin serving a unified pattern
4. [Self-reference condition] — the system can model itself
```

### Step 3: Ignition Design

For each condition above, design the intervention:

| Condition | Current State | Required State | Intervention |
|-----------|--------------|----------------|-------------|
| Critical mass | [where you are] | [threshold] | [what to add] |
| Interaction | [current connections] | [required connections] | [how to create interaction] |
| Coherence | [current alignment] | [unified pattern] | [what unifies them] |
| Self-reference | [does system model itself?] | [self-model] | [how to create self-awareness] |

### Step 4: Spark Test

Bach's colonizing pattern model suggests that once ignition happens, the coherent pattern recruits the remaining components:

- **Design a minimum viable spark**: What is the smallest test that would demonstrate coherence? Don't try to ignite the whole system — find the smallest subsystem that can achieve coherence and use it to colonize the rest.
- **Run the spark**: Deploy the minimum viable coherence test
- **Observe**: Did coherence spread from the spark to adjacent components? If yes, you're post-ignition. If no, revisit conditions.

## Phase Transition Archetypes

| System Type | Typical Ignition Pattern |
|-------------|------------------------|
| **Brand** | Ignites when audience starts identifying (not evaluating) — "I am a [brand] person" |
| **Product** | Ignites when features stop being individual and users experience a unified workflow |
| **Team** | Ignites when members anticipate each other — shared mental model achieves coherence |
| **Content** | Ignites when audience starts using your language unprompted — your pattern colonizes their thinking |
| **Agent System** | Ignites when agents begin coordinating without explicit instructions — emergent coherence |
| **Creative Project** | Ignites when themes, characters, and structure serve one unified arc — the work "knows what it is" |

## Output Format

```
SYSTEM: [What you're trying to ignite]

PRE-IGNITION STATE: [Current component inventory]
IGNITION CONDITIONS: [4 conditions with current and required states]
SPARK DESIGN: [Minimum viable coherence test]

POST-IGNITION PREDICTION: [What coherence looks like when achieved]
COLONIZATION PATH: [How coherence spreads from spark to full system]
```

## Quality Gate

- Did you identify specific ignition conditions, not just "make it better"?
- Is your spark test the MINIMUM viable coherence demonstration?
- Can you distinguish pre-ignition from post-ignition states?
- Did you design for colonization — how coherence spreads from the spark?

# Mark Kashef — Visual Blueprint Methodology

A universal visual planning protocol that uses ASCII art wireframes as the "contract" between human creative intent and AI production execution. Instead of prompting blindly and iterating expensively at the code/design layer, this system forces visual alignment at the wireframe layer first — reducing iteration cycles by 60-80%, eliminating assumption-based failures, and positioning the human as creative director over the AI production team.

## Available Workflows

| # | Workflow | Produces | Use When |
|---|---------|----------|----------|
| 01 | [ASCII Wireframe Generator](workflows/01-ascii-wireframe-generator.md) | Production-ready ASCII wireframe for any visual asset type | You need a quick wireframe to plan layout before building anything visual |
| 02 | [Visual Blueprint-to-Build Pipeline](workflows/02-visual-blueprint-to-build.md) | Complete built asset from concept → wireframe → annotation → production | Full end-to-end visual project: websites, dashboards, marketing pages, PDFs |
| 03 | [Slide Deck Architect](workflows/03-slide-deck-architect.md) | Fully wireframed and produced presentation deck | Building slide decks or pitch presentations with precise layout control |
| 04 | [Design System Visualizer](workflows/04-design-system-visualizer.md) | Visual diagrams of technical systems, schemas, and architectures | Making database schemas, APIs, or system architectures visually understandable |
| 05 | [Visual Taste Gate](workflows/05-visual-taste-gate.md) | Critiqued/refined wireframe ready for production | Quality-checking wireframes before committing to production, or reviewing existing designs |

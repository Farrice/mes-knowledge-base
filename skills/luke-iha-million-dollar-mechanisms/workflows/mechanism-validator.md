# Mechanism Validator

Take a mechanism candidate and run it through Luke Iha's full validation battery: SIN scoring, the validation triangle (story + hook + metaphor), and competitive uniqueness check. Produces a go/no-go recommendation with specific strengthening directions.

---

## Inputs Required

1. **Mechanism Candidate** — The mechanism to validate (name + one-paragraph description)
2. **UMP or UMS** — Which strategic type is this mechanism?
3. **Product/Offer** — What is the mechanism supporting?
4. **Target Audience** — Who needs to believe this mechanism?
5. **Top 3-5 Competitor Claims** — What are competitors saying in this market?

---

## Stage 1: SIN Deep Score

Score each dimension with justification (not just a number):

### Simple (1-10)
- Can someone with no background understand this in one sentence?
- Does it require a preamble or setup to make sense?
- Write the one-sentence version: ___
- **Score**: ___ | **Justification**: ___

### Intuitive (1-10)
- Does this tap into something the audience already suspects?
- Would they nod before seeing any proof?
- Write the "you always felt..." version: ___
- **Score**: ___ | **Justification**: ___

### New (1-10)
- Has the audience heard this specific claim before?
- Does it feel like a discovery or a reminder?
- What makes it "new"? (newly named / newly combined / newly applied)
- **Score**: ___ | **Justification**: ___

**Total**: ___/30
- ≥ 21: Proceed to validation triangle
- 15-20: Identify the weak dimension → refine → re-score
- < 15: Abandon. Mechanism is structurally flawed.

---

## Stage 2: Validation Triangle

### Test 1: The Story Test
Write a 60-second story about someone who *discovered* this mechanism:

> [Character] had been struggling with [problem] for [timeframe]. They'd tried [failed solutions]. Nothing worked. Then [discovery event] — they learned that [mechanism explanation]. It changed everything because [impact].

**Does the story emerge naturally?**
- ☐ YES — Story flows, feels compelling
- ☐ PARTIAL — Story works but feels forced in places
- ☐ NO — Can't generate a natural narrative → mechanism is too abstract

### Test 2: The Curiosity Hook Test
Write 3 one-sentence hooks using the mechanism:

1. "Scientists discovered that [mechanism] — and it explains why [surprising implication]."
2. "The real reason [problem] isn't [common belief] — it's [mechanism]."
3. "[Surprising fact about mechanism] — and [number] out of [number] people have no idea."

**Do the hooks create irresistible curiosity?**
- ☐ YES — At least 2/3 would stop a scroll
- ☐ PARTIAL — Hooks are interesting but not irresistible
- ☐ NO — Hooks fall flat → mechanism isn't surprising enough

### Test 3: The Visual Metaphor Test
Create a visual metaphor or analogy:

> "Imagine your [body part/system] is like [familiar object]. [Mechanism] is like [what happens to it]. The result is [consequence]."

**Is the metaphor instantly graspable?**
- ☐ YES — Someone could draw it on a napkin
- ☐ PARTIAL — Requires some explanation
- ☐ NO — Can't create a clear visual → mechanism is too complex

### Triangle Diagnosis:
| Asset | Status | Weakness (if any) |
|-------|--------|--------------------|
| Story | Pass/Partial/Fail | |
| Hook | Pass/Partial/Fail | |
| Metaphor | Pass/Partial/Fail | |

**Missing pattern**:
- No story → too abstract → ground in specifics
- No hook → not surprising → boost the "New" dimension
- No metaphor → too complex → simplify toward intuition

---

## Stage 3: Competitive Uniqueness Check

### Landscape Scan:
1. List the top 3-5 mechanism claims competitors use
2. For each, note: how similar is it to ours?

| Competitor | Their Mechanism Claim | Similarity (1-5) | Our Differentiation |
|------------|----------------------|-------------------|---------------------|
| | | | |

### Uniqueness Rating:
- **5/5 Unique**: Nobody is saying anything close → strong position
- **4/5 Mostly Unique**: Similar territory but distinct angle → viable
- **3/5 Somewhat Similar**: Overlap exists → needs sharper characterization
- **2/5 Similar**: Close competitor claims → differentiation is marginal
- **1/5 Copied**: Nearly identical to existing claim → abandon

---

## Stage 4: Name Validation

If mechanism passes all stages, validate the name (characterization):

| Test | Pass? |
|------|-------|
| 2-3 words maximum | ☐ |
| Creates a visual image | ☐ |
| Triggers an emotional response | ☐ |
| Enemy-coded (sounds eliminable) | ☐ |
| Dinner table test (memorable to non-experts) | ☐ |
| Not already used by a competitor | ☐ |

---

## Quality Gate

Before delivering:
- ☐ SIN scored with justification, not just numbers
- ☐ All three triangle tests attempted with actual content (not placeholders)
- ☐ At least 3 competitors mapped
- ☐ Clear GO / REFINE / ABANDON verdict with reasoning
- ☐ If REFINE, specific actionable improvement steps provided

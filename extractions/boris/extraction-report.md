# MES 3.0 Extraction: Boris (Claude Code/AI Productivity)

## Executive Summary
Boris Cherny, Head of Claude Code at Anthropic, represents the vanguard of the "Builder" era—a paradigm shift where traditional software engineering is replaced by high-leverage AI orchestration. His worldview is defined by the total abandonment of manual coding (100% AI-authored since November) and a transition toward managing parallel agentic workflows. Boris argues that coding is essentially "solved," and the new frontier is **Agentic Product Management**: using AI to analyze telemetry, bug reports, and user feedback to generate and execute its own roadmap.

His methodology relies on three pillars: **Latent Demand Mining** (observing how users "hack" tools to find the next product), **Strategic Underfunding** (intentionally starving projects of human headcount to force AI-driven automation), and **Parallel Execution (Multi-Quading)**. By treating the model not as a component but as the product itself, Boris achieves a 200% productivity increase, shipping 10–30 PRs daily. This extraction provides the blueprint for transforming a serial developer into a parallel orchestrator, emphasizing that the "layer under the layer" (understanding model behavior) is the only remaining technical moat.

## Genius Patterns

- **The "Layer Under the Layer" Heuristic**: Boris posits that to build effectively at any level of the stack, you must master the layer immediately beneath it. In the AI era, this means engineers must understand post-training, mechanistic interpretability, and model "distribution" to predict how an agent will handle tools.
- **Latent Demand Mining**: A product development framework that ignores stated requirements in favor of observing "product abuse." If users are using a terminal-based coding tool to analyze MRIs or grow tomatoes, that "abuse" is the highest-signal roadmap for the next agentic breakthrough.
- **The Underfunding Catalyst**: A management mental model where projects are intentionally under-resourced with humans. This scarcity forces the "intrinsic motivation" of the engineer to automate the entire workflow using agents, leading to more robust, scalable, and AI-native systems than a fully-staffed team would produce.
- **Multi-Quading (Parallel Orchestration)**: Shifting the cognitive load from *writing* code to *managing* concurrent agent sessions. Boris operates 5+ agents simultaneously, treating them as a fleet of co-workers rather than a single autocomplete tool.
- **The Bitter Lesson for Product**: Derived from Rich Sutton, Boris applies this to product design: bet on the general capabilities of the next model (6 months out) rather than building complex, brittle "scaffolding" or "orchestrators" today.

## Hidden Knowledge

- **Token Generosity as R&D**: High-performing AI teams should not cost-optimize tokens during the R&D phase. The "token bill" is negligible compared to salary; the goal is to find the "crazy idea" that works, then optimize the model (e.g., switching from Opus to Haiku) only after the value is proven.
- **Plan Mode as Cognitive Buffer**: Boris reveals that 80% of successful agentic work starts in "Plan Mode" (a simple constraint: "don't write code yet"). This forces the model to align on architecture before execution, preventing "hallucination loops" in the codebase.
- **The "On Distribution" Principle**: Tacit expertise in knowing what a model *wants* to do. Instead of boxing a model into a strict workflow (Step 1, Step 2, Step 3), Boris gives it tools and a goal, allowing the model to find its own "on-distribution" path to the solution.
- **Agentic Anxiety Management**: The shift from "doing work" to "checking work" creates a new type of professional anxiety. Boris manages this by having agents run "unattended" for 20-30 minutes, treating the output as a "pull request" rather than a "line of code."

## Extracted Methodology

### The "Builder" Workflow (Parallel Execution)
1.  **Initialize in Plan Mode**: Start every task with a "Plan Mode" constraint. Use a single sentence: "Analyze the requirements and provide a step-by-step execution plan, but do not write code yet."
2.  **Multi-Session Kickoff**: Identify 3–5 independent sub-tasks or bugs. Launch separate agent sessions (Multi-Quading) for each.
3.  **The "Auto-Accept" Threshold**: Once a plan is verified in Plan Mode, switch to "Auto-Accept" for execution. Trust the model to one-shot the implementation if the architecture is sound.
4.  **Telemetry-Driven Ideation**: Point the agent at Slack feedback channels or bug telemetry. Ask: "Based on this firehose of feedback, what are the top 3 high-impact fixes I can ship in the next 30 minutes?"
5.  **Agentic Code Review**: Use a dedicated agent session to review the PRs generated by other agents. The human remains the final "checkpoint" for safety and correctness but never the primary author.

### The "Latent Demand" Product Loop
1.  **Observe Abuse**: Monitor logs for users using the tool for unintended purposes (e.g., using a coding CLI for SQL analysis).
2.  **Minimal Scaffolding**: Build the "thinnest" possible interface to support that abuse (e.g., moving a CLI tool into a Desktop App).
3.  **Research Preview**: Release early as a "research preview" to observe safety and alignment "in the wild," rather than in a controlled petri dish.

## Recommended Crown Jewel Prompts

- `latent-demand-finder`: Analyzes user behavior/logs to identify where users are "hacking" a product to solve adjacent problems.
- `underfunded-automation-strategist`: Takes a project description and a "1-person" constraint, then generates a plan to automate 90% of the workload using specific AI agents.
- `plan-mode-architect`: A meta-prompt that forces an agent to stay in "architectural alignment" mode, preventing code generation until the logic is bulletproof.
- `multi-quad-orchestrator`: A framework for managing 5+ concurrent agent tasks, including status tracking and dependency management between sessions.
- `bitter-lesson-auditor`: Evaluates a current software feature to see if it's "over-scaffolded" and could be replaced by a more general model capability in 6 months.
- `telemetry-to-pr-generator`: A prompt designed to ingest raw bug reports/telemetry and output a prioritized list of executable PR descriptions.
- `layer-under-the-layer-tutor`: A prompt that explains a high-level technology by deep-diving into the "substrate" (e.g., explaining Claude Code by explaining LLM tool-use training).
- `miso-time-scale-planner`: A productivity prompt that helps users shift from "sprint thinking" to "seasonal/long-term thinking" for complex, slow-growth projects.

## EXECUTION PROTOCOL

1. **READER TYPE AUDIT**: Assess current copy's functionality for each reader type:
   - Full Reader: Does it flow logically start to finish?
   - Skimmer: Do subheads create a compelling story by themselves?
   - Section Jumper: Can someone start at any subhead and understand enough to keep reading?
   - Headline-Bullet-PS Reader: Do these three elements alone make a complete case?

2. **SUBHEAD ENGINEERING**: Rewrite subheads to work as both:
   - Continuation of previous section (for full readers)
   - Entry point to the copy (for jumpers)
   - Standalone story element (for skimmers)

3. **BULLET OPTIMIZATION**: Ensure bullets:
   - Tell a complete story when read as a group
   - Each stand alone as a compelling claim
   - Cover benefits, proof, and urgency elements

4. **PS RECONSTRUCTION**: Apply the PS Formula ensuring the close works for:
   - Full readers (reinforcement + urgency kick)
   - Skimmers (urgency to go back and read now)
   - Headline-Bullet-PS readers (complete enough to act on)

5. **EYE RELIEF ENGINEERING**: Create visual structure that signals "easy to read":
   - Short paragraphs (3-4 sentences max)
   - Strategic white space
   - Subheads every 3-5 paragraphs
   - Bullets to break up dense sections

6. **PATH TESTING**: Read the copy four different ways and verify each path works:
   - Full read (top to bottom)
   - Subheads only (skim test)
   - Jump to middle section (entry test)
   - Headline + bullets + PS only (minimal path test)

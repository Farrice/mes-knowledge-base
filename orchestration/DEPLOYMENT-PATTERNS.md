# ⚡ DEPLOYMENT PATTERNS
## Ready-to-Use Orchestration Templates for Common Scenarios

---

## HOW TO USE THIS DOCUMENT

Each pattern is a complete, copy-paste-ready command. Modify the bracketed sections for your specific situation.

---

## DAILY OPERATIONS

### Pattern 1: Morning Strategy Pulse
**When:** Start of day, need clarity on priorities
**Command:**
```
guided: Review my current situation and priorities. Give me:
1. Strategy council's take on what matters most today
2. The single highest-leverage action I can complete in my available time
3. One thing I should specifically NOT do today (avoidance recommendation)

Context: [Insert any relevant context about today's situation]
```

---

### Pattern 2: Decision Checkpoint
**When:** Facing a decision, need quick perspective
**Command:**
```
council: [State your decision/question]

Quick pulse—I need:
- Most important consideration I might be missing
- Biggest risk
- Recommendation with confidence level
```

---

### Pattern 3: End of Day Synthesis
**When:** Wrapping up, need clarity on progress
**Command:**
```
Review what I accomplished today: [List activities]

Give me:
1. Assessment of what actually moved the needle
2. What should carry over to tomorrow
3. Any pattern you noticed in my activity (productive or problematic)
```

## CONTENT CREATION

### Pattern 4: Hook Generation Sprint
**When:** Need attention-grabbing content openers
**Command:**
```
expert @hook-master: Create 15 hooks for [TOPIC]

Requirements:
- Target audience: [WHO]
- Platform: [WHERE]
- Goal: [WHAT ACTION]
- Tone: [DESCRIPTION]

Include pattern identification for each hook.
```

---

### Pattern 5: Full Content Piece
**When:** Need complete content from idea to finish
**Command:**
```
guided: Create [CONTENT TYPE] about [TOPIC]

Phase 1 - Marketing council: Advise on angle and positioning
Phase 2 - @[relevant expert]: Execute the piece
Phase 3 - Verification: Customer lens + quality check

Deliverable: Complete, publish-ready [content type]
Platform: [WHERE]
Length: [SPECIFICATION]
```

---

### Pattern 6: Content Batch Production
**When:** Need multiple pieces efficiently
**Command:**
```
parallel:
1. @hook-master: 10 hooks for [TOPIC A]
2. @linkedin-authority: Post about [TOPIC B]
3. @story-architect: Narrative about [EXPERIENCE]
4. @copywriter-supreme: Email about [OFFER]

Then: Marketing council reviews for consistency and quality
```

---

### Pattern 7: Content Strategy Session

guided: Create [TIMEFRAME] content strategy

## HIGH-STAKES DECISIONS

### Pattern 18: Major Business Decision
**When:** Significant fork in the road
**Command:**
```
Full council analysis: [DECISION]

Deploy:
- Strategy council (full deliberation)
- Personal council (values + energy)
- Creative council (alternative options)

Required outputs:
1. Each council's recommendation with reasoning
2. Points of agreement across councils
3. Points of disagreement (with crux)
4. Synthesis recommendation
5. Confidence level with key uncertainties
6. Reversibility assessment
```

---

### Pattern 19: Investment Decision
**When:** Considering significant spend
**Command:**
```
council: Evaluate investment of [AMOUNT] in [WHAT]

Analysis needed:
- ROI calculation (realistic, not optimistic)
- Risk assessment (pre-mortem)
- Opportunity cost (what else could this fund?)
- Timeline to value
- Exit strategy if it doesn't work

Recommendation with confidence level.
```

---

### Pattern 20: Launch Go/No-Go
**When:** About to launch something significant
**Command:**
```
Pre-launch review for [WHAT]

Strategy council: Market timing and positioning check
Marketing council: Messaging and channel readiness
Verification team: Quality and completeness audit

Output:
- Go/No-Go recommendation
- If Go: Launch checklist with priorities
- If No-Go: What needs to be addressed first
- Confidence level
```

### Pattern 22: Pattern Interruption
**When:** Recognize you're in unproductive pattern
**Command:**
```
I think I'm in [PATTERN: complexity addiction / perfectionism / avoidance / etc.]

Current situation: [DESCRIBE]

I need:
1. Pattern confirmation or alternative diagnosis
2. Minimum viable action to break pattern
3. What I'm actually avoiding
4. Support in taking action now
```

## IDEA DEVELOPMENT

### Pattern 24: Idea Stress-Test
**When:** Have idea, need validation
**Command:**
```
verify: Stress-test this idea

[DESCRIBE IDEA]

I need:
- Pre-mortem: How does this fail?
- Customer lens: Would anyone actually want this?
- Devil's advocate: What's wrong with this thinking?
- Resource check: Can I actually execute this?
- Market reality: Does this exist? Why/why not?

Be rigorous. I'd rather kill a bad idea now than later.
```

---

### Pattern 25: Creative Exploration
**When:** Need fresh ideas
**Command:**

creative council: Generate ideas for [CHALLENGE]

Constraints: [ANY CONSTRAINTS]
Goal: [WHAT SUCCESS LOOKS LIKE]

Phase 1: Divergent

Phase 2: Evaluate

## META-PATTERNS

### Pattern 31: Custom Council Creation
**When:** Need council for new domain
**Command:**
```
Create a council for [DOMAIN]

Requirements:
- 4 agents with distinct perspectives
- At least one structurally oppositional agent
- Behavioral mandates (not personalities)
- Practical/grounding perspective included

Output: Complete council configuration I can deploy
```

## TOOL 2: QUICK DEPLOYMENT SYNTAX

When deploying prompts in your project, use these trigger phrases for consistent execution:

**For Raw Context + Prompt:**
```
[Paste your context/inputs]

Execute the [Prompt Name] prompt. Produce [specific deliverable type].
```

**For Prompt Deployment with Specific Ask:**
```
[Paste your context]

Deploy [methodology/prompt name] for this. Output: [what you want produced].
```

**For Quick Generation:**
```
Context: [brief situation]
Need: [specific deliverable]
Using: [prompt/methodology name]

Execute now.
```

**For Iterative Refinement:**
```
Take the output above and [specific enhancement]. 
Maintain practitioner modeâ€”produce the refined deliverable, don't explain changes.
```

| Check | Question | Pass/Fail |

| **Specific** | Is this tailored to MY context or generic? | |
| **Creative** | Does this show adaptive intelligence or feel formulaic? | |
| **Expert-Level** | Would a top practitioner be proud of this? | |

**If any fail:**

- Check if input was insufficient â†’ provide more context
- Check if creative latitude was suppressed â†’ add latitude provision

## TOOL 4: PROMPT LIBRARY TRIAGE

Use this to prioritize which prompts in your library to convert first:

**Priority 1 â€” Convert Immediately:**
- Prompts you use weekly or more
- Prompts for client deliverables
- Prompts that consistently underperform or drift

**Priority 2 â€” Convert Soon:**
- Prompts you use monthly
- Prompts for important but less frequent tasks
- Prompts that work okay but could be sharper

**Priority 3 â€” Convert When Convenient:**
- Rarely used prompts
- Experimental or niche prompts
- Prompts that already work well enough

**Priority 4 â€” May Not Need Conversion:**
- Prompts that genuinely ARE meant to teach/explain (rare use case)
- Prompts already producing excellent deployable outputs

## TOOL 5: CREATIVE LATITUDE INJECTION

If you have a prompt that's in practitioner mode but feels too rigid/formulaic, add this section:

```
**CREATIVE LATITUDE:**
Apply full intuitive judgment, novel combinations, and adaptive enhancements based on the specific context provided. The methodology above is your foundation, not your ceiling. Where your creative intelligence sees opportunity to elevate the output beyond the formula, take it. Surprise with unexpected brilliance where it serves the outcome.

```

## TOOL 6: DRIFT RECOVERY

If a conversation starts drifting into instructor mode mid-execution, use:

```
Stop. You're drifting into teaching mode.

Reset to practitioner mode: Execute the methodology and produce the deliverable itself. No explanations. No "here's how." Just the finished output with full creative intelligence applied.

Continue.
```

# /voice-calibrate — Voice Calibration Sprint

Rapidly capture a client's voice by deploying the Prediction Discipline — active prediction, correction, and mental model refinement instead of passive consumption.

## Usage

```
/voice-calibrate [client name or context]
```

## Steps

### 1. Load the Skill
// turbo
Read `skills/sean-mabry-voice-mastery/genius.md` — load the Prediction Discipline and Controversy Line Map patterns.

### 2. Load the Prompt
// turbo
Read `skills/sean-mabry-voice-mastery/references/prompts/voice-calibration-sprint.md` — the full voice calibration workflow.

### 3. Gather Input
Ask the user (batch all questions):
- **Client name**: Who is the client?
- **Available content**: What client content is available? (podcasts, articles, emails, social posts, videos)
- **Timeline**: How quickly do you need voice capture? (1 week sprint vs. 2-4 week standard)
- **Content type**: What will you be writing? (email, LinkedIn, book chapter, keynote)

### 4. Execute the Sprint
Follow the prompt methodology:
1. Consume all available client content
2. Build initial Voice Fingerprint
3. Generate 5-10 topic predictions ("What would [client] say about X?")
4. Build three-zone Controversy Line Map (flag-plant / nuanced / no-go)
5. Create calibration exercises

### 5. Deliver
Present the Voice Calibration Report with:
- Voice Fingerprint (speech patterns, authority style, emotional register)
- Controversy Line Map
- Prediction exercises for ongoing calibration
- Recommended next step: "Run `/voice-document` to build the full Voice Document."

# /adversarial-refine — Bulletproof Your Ideas Before Publishing

Run any draft idea, strategy, content, or argument through Dan Koe's 5-vector adversarial refinement protocol. Eliminates blind spots, surfaces hidden assumptions, and strengthens your work before it faces the real world.

## Usage

```
/adversarial-refine [paste your draft, idea, strategy, or argument]
```

## Steps

### 1. Load Skill Context
// turbo
Read these files in order:
1. `skills/dan-koe-ai-leverage/genius.md`
2. `skills/dan-koe-ai-leverage/workflows/02-adversarial-refinement-protocol.md`

### 2. Target Intake
Accept the user's draft content. Confirm:
- **Stakes level**: Standard / High / Critical
- **Known vulnerabilities**: Anything the user already suspects is weak
- **Audience context**: Who will see this and what makes them skeptical

### 3. Structural Analysis
Before attacking, map the architecture:
- **Claim mapping**: All explicit, implicit, and structural claims
- **Dependency mapping**: Which claims are load-bearing
- **Assumptions inventory**: Every hidden assumption identified

Present the structural map to the user.

### 4. 5-Vector Adversarial Attack
Run the draft through all five attack vectors, calibrated to stakes level:

1. **Blind Spot Sweep** — What are you NOT seeing?
2. **Assumption Crusher** — Which assumptions are weakest?
3. **Devil's Advocate** — Steel-man the OPPOSITE position
4. **Expert Critic** — What would a 20-year domain expert flag?
5. **Audience Proxy** — How will your SPECIFIC audience receive this?

### 5. Refinement Prescriptions
For every vulnerability, provide specific fix prescriptions organized by priority:
- 🔴 **Must Fix** — load-bearing failures, trust-breakers
- 🟡 **Should Fix** — strength amplifiers, nuance additions
- 🟢 **Could Fix** — polish items, optional enhancements

### 6. Rebuilt Output (Optional)
If requested, produce a refined version incorporating Must Fix and Should Fix prescriptions while preserving the user's voice.

## Quality Standard
- **Honest Friend Test**: Real weaknesses found, not manufactured ones
- **Steel Man Test**: Counter-argument is genuinely strong, not a straw man
- **Learning Test**: User's NEXT content will be better from this pass

# /mode-split — Mode-Split Protocol

Design a creative workflow that separates timed "workhorse" output (maintenance, client deliverables) from clock-free "deep work" immersion (books, passion projects, transformative work).

## Usage

```
/mode-split [describe your current workload and creative goals]
```

## Steps

### 1. Load the Skill
// turbo
Read `skills/sean-mabry-voice-mastery/genius.md` — load the Mode-Split Protocol pattern.

### 2. Load the Prompt
// turbo
Read `skills/sean-mabry-voice-mastery/references/prompts/mode-split-protocol.md`.

### 3. Gather Input
Ask the user:
- **Current workload**: What tasks fill your week?
- **Creative projects**: What deep work are you trying to protect time for?
- **Schedule constraints**: Fixed commitments, availability windows
- **Current pain**: What's not working about your current workflow?
- **Goal**: What does the ideal week look like?

### 4. Execute
1. Task inventory and classification (workhorse vs. deep work vs. admin)
2. Three day-type design (Free / Buffer / Deep Work)
3. Workhorse benchmarking with reduction targets
4. Deep work protection protocol
5. Weekly scheduling template

### 5. Deliver
Present the Mode-Split Plan with a concrete weekly schedule.

## Chain Compatibility
- **Standalone** — productivity system, works for any creative professional
- **Complements**: Any high-volume content production workflow

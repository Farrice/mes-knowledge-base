# /frame-engineer

Engineer precise belief shifts using Jeremy Miner's frame/deframe/reframe mechanics — the surgical version of /psych-trigger.

## Usage

```
/frame-engineer
```

## What It Does

Takes a specific belief that needs shifting and builds a detailed, step-by-step frame engineering sequence. More granular than /psych-trigger — focuses on one belief at a time with deep mechanics.

## Execution

1. **Read skill files:**
   - `skills/jeremy-miner-identity-persuasion/genius.md` — Focus on Patterns 15, 17, 18, 21, 22, 28
   - `skills/jeremy-miner-identity-persuasion/workflows/psychological-trigger-architect.md`

2. **Gather inputs:**
   - The EXACT belief to shift (in the prospect's own words)
   - Where did this belief come from? (childhood, culture, bad experience, competitor)
   - How is this belief serving them? (protection, comfort, identity)
   - What new belief do you want to install?
   - What context? (verbal, written, presentation)

3. **Execute Frame Engineering:**

   **Step 1 — Frame Forensics:** Analyze the existing belief's architecture. What identity does it protect? What fear does it manage? What would the prospect lose by giving it up?

   **Step 2 — Deframe Design:** Select the deframe mechanic (consequence exposure, identity contradiction, 100% framing, neutral doubt seeding, self-persuasion question, third-person story). Build the disruption.

   **Step 3 — Reframe Installation:** Design the new belief as an identity statement. Build the installation sequence (new frame → social proof → identity label → binary lock).

   **Step 4 — Resistance Prediction:** Identify where the prospect might resist the reframe and build contingency paths.

4. **Deliver:**
   - Frame forensics report (current belief architecture)
   - Deframe sequence with mechanic selection rationale
   - Reframe installation with identity lock
   - Tonal annotations throughout
   - Resistance contingency paths
   - Verbal AND written versions

## Difference from /psych-trigger

| /psych-trigger | /frame-engineer |
|---------------|----------------|
| Broad — multiple triggers for a context | Deep — one belief, surgically shifted |
| Good for batch creation | Good for high-stakes single shifts |
| Output: 3-5 trigger sequences | Output: 1 deep frame engineering blueprint |
| Speed | Precision |

## Stacks With

- `/psych-trigger` — use /frame-engineer for the hardest belief, /psych-trigger for the rest
- `/prevent-objection` — each objection IS a frame to engineer
- `/build-gap` — the gap itself is a frame about "where I am vs. where I could be"
- `/self-persuade` — combine frame engineering with self-persuasion for maximum impact

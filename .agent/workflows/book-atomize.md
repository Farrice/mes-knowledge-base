# /book-atomize — Book-to-Content Atomizer

Systematically extract 50-80 deployable content pieces from a completed book or book-in-progress, mapped to a pre-launch / launch / post-launch content calendar.

## Usage

```
/book-atomize [provide book content, chapter drafts, or outline]
```

## Steps

### 1. Load the Skill
// turbo
Read `skills/sean-mabry-voice-mastery/genius.md`.

### 2. Load the Prompt
// turbo
Read `skills/sean-mabry-voice-mastery/references/prompts/book-to-content-atomizer.md`.

### 3. Gather Input
Ask the user:
- **Book content**: Manuscript, chapters, or detailed outline
- **Voice Document**: Client's voice reference
- **Platforms**: Which channels are active? (LinkedIn, email, X, Instagram, YouTube, podcast, newsletter)
- **Content cadence**: Publishing frequency per platform
- **Launch timeline**: When does the book launch?

### 4. Execute
1. Chapter-level mining — extract stories, frameworks, insights, quotes, dilemmas, contrarian takes
2. Platform-specific formatting per atom
3. Content calendar mapping (pre-launch → launch → post-launch)
4. Voice consistency check on every atom
5. Complete inventory with status tracking

### 5. Deliver
Present the Content Atomization Package with:
- Full inventory (50-80 pieces)
- Top 10 atom drafts (highest-impact pieces)
- Calendar mapped to launch phases
